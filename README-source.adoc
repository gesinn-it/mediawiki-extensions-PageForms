Turn your MediaWiki into a structured data platform — without writing a single line of code.

Page Forms lets wiki editors create and edit pages through clean, intuitive forms instead of raw wikitext.
Admins define forms once; editors fill them in from then on.
The result: consistent data, fewer errors, and a wiki that non-technical users actually enjoy working with.

[NOTE]
====
This is a maintained distribution of the upstream https://www.mediawiki.org/wiki/Extension:Page_Forms[Page Forms] extension, hosted on GitHub for easier contribution.
It ships with CI on every pull request, expanded test coverage, and is actively developed with a focus on https://www.semantic-mediawiki.org[Semantic MediaWiki] integration.
Compatible with the upstream extension, with two intentional exceptions: support for Cargo and PageSchemas has been dropped.
====

== What Page Forms does

=== Forms for editors

Editors get a dedicated *Edit with form* tab on every page that has a form assigned.
Instead of editing raw wikitext, they fill in clearly labelled fields — text inputs, dropdowns, date pickers, tag tokens, file uploads, interactive maps, and more.
The extension writes the correctly structured wikitext back to the page automatically.

=== Full control for admins

Form definitions live in the `Form:` namespace as ordinary wiki pages.
A form definition specifies which templates and fields appear, what input type each field uses, which values are allowed, and how the resulting page is named.
No deployment required — editing a form definition takes effect immediately.

=== Structured data with Semantic MediaWiki

When used with https://www.semantic-mediawiki.org[Semantic MediaWiki], Page Forms becomes a complete data-entry layer for your wiki knowledge base:
fields are wired directly to SMW properties, autocompletion draws from live property values, and the helper page `Special:CreateClass` scaffolds the entire template/form/category/property stack in one step.

=== Power features

* *Multiple-instance templates* — let editors add, remove, and reorder repeating data blocks (skills, publications, addresses, …) within a single form
* *Spreadsheet and calendar views* — edit multiple template instances as a grid or as calendar events
* *`#autoedit`* — create or update pages silently in the background from any wiki page, without opening a form
* *Query forms* — build search interfaces with `Special:RunQuery` that let users filter and explore data through form fields
* *`#formredlink`* — red links that go directly to the right form for creating that page

|===
| MediaWiki | >= 1.39

== Quick start

[source,php]
----
// LocalSettings.php
wfLoadExtension( 'PageForms' );
----

Then open `Special:CreateClass` on your wiki to scaffold your first form-backed data structure in minutes.

Full installation instructions and the complete documentation are in link:docs/index.adoc[docs/].
