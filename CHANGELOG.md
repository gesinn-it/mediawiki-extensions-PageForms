### Fixed
- `PFFormPrinter::formHTML()`: submitting a field via the autoedit `+`/`-` append/prepend modifier (e.g. `TemplateName[FieldName+]=value`) no longer emits an "Undefined array key" warning when the target page has no existing template call for that field. `PFTemplateInForm::getValuesFromPage()` only contains keys explicitly present on the page (`mValuesFromPage` defaults to `[]`), so the unguarded `getValuesFromPage()[$field_name]` access failed for new fields/pages; now defaults to `''` via `?? ''`, the same value the modifier already degraded to on failure ([#120](https://github.com/gesinn-it/mediawiki-extensions-PageForms/issues/120))
- `PFFormPrinter::formHTML()`: an unclosed `{{{` tag (no matching `}}}` anywhere in its section) is now reported with an actionable `MWException` instead of being silently dropped from the output. `strpos()` returning `false` for the missing `}}}` was coerced to `3` by PHP's `false + 3` arithmetic, corrupting the `substr()` extraction of the bracketed string so the tag vanished with no error. Also guarded the "more than 3 closing braces" scan (`while ( $section[$brackets_end_loc + 3] == "}" )`) with `isset()` so a tag whose `}}}` lands on the last three characters of the section can no longer read past the end of the string ([#119](https://github.com/gesinn-it/mediawiki-extensions-PageForms/issues/119))
- `PFFormPrinter::formHTML()`: an empty `{{{}}}` tag in a form definition no longer hangs the request. `PFUtils::getFormTagComponents('')` returns `[]`, and the tag-processing loop's `continue` on that case never advanced `$start_position`, so the same empty tag was found at the same position forever; changed to `break`, matching the equivalent guard already present in `src/FormDefParser.php`. The sibling `splitFormDefIntoSections()` helper had the identical unguarded `$tag_components[0]` access (no `continue`/`break` at all), which threw an "Undefined array key" error for the same input before the loop above was ever reached; it now skips tag-boundary detection for a tag with no components instead of indexing into the empty array ([#118](https://github.com/gesinn-it/mediawiki-extensions-PageForms/issues/118))
